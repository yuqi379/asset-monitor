# 资产监控系统前端重构计划

## Goal

将资产监控系统前端从 `jQuery + DataTables` 重构为 `Vue3 + Vite + Element Plus`，后端 Flask API 不变。

## 当前状态

```
asset_monitor/
├── api_server.py          # Flask API（不动）
├── web/
│   └── index.html         # jQuery + DataTables 单文件（约500行）
├── core/                  # Python 后端逻辑（不动）
├── platforms/             # Hunter/GitHub 平台对接（不动）
└── database/             # SQLite 数据库（不动）
```

## 重构策略

**保留 `web/` 目录作为回滚备底，新建 `web-vue/` 目录全新开发。**

Flask api_server.py 静态文件指向改为 `web-vue/dist/`，完成后删 `web/`。

---

## 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 (Composition API) | 响应式，组件化 |
| 构建工具 | Vite | 快，配置简单 |
| UI 组件库 | Element Plus | 中文文档完善，后台管理组件齐全 |
| HTTP 客户端 | Axios | 统一拦截器管理 |
| 路由 | Vue Router | 后续多页面扩展 |
| 状态管理 | Pinia | 轻量，后续页面多了再用 |

## 实施步骤

### Step 1 — 初始化项目

```bash
cd /home/simple27/.hermes/scripts/asset_monitor/
mkdir -p web-vue && cd web-vue
npm create vite@latest . -- --template vue
npm install element-plus axios vue-router pinia
```

### Step 2 — 配置 Vite 代理

`vite.config.js` 配置代理，将 `/api` 请求转发到 Flask 端口（5000），解决开发环境跨域。

### Step 3 — 迁移资产列表页面

`views/AssetList.vue` 还原现有功能：
- 表格展示（IP/端口/域名/标题/企业/省份/城市/平台/状态/查询条件/发现时间）
- 自定义分页栏（每页条数下拉 + 页码按钮 + 上一页/下一页 + 共X条）
- 搜索框（实时过滤）
- 开始检测按钮（调用 `/api/detect`）
- 导出Excel按钮（DataTables buttons 迁移）
- 统计卡片（总数/平台数/IP数/省份数/待检测/异常）

### Step 4 — 适配现有 API

Flask API 保持不变，前端 axios 对接：

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/assets` | GET | 获取资产列表 |
| `/api/detect` | POST | 触发检测 |
| `/api/collect` | POST | 上传关键词Excel触发采集 |

### Step 5 — 打包并切换

```bash
npm run build   # 输出到 dist/
```

修改 `api_server.py` 静态文件路径指向 `web-vue/dist/`。

## 验证步骤

1. `npm run dev` 启动 Vite 开发服务器，浏览器访问 `http://localhost:5173`
2. 确认表格数据正常加载（调用 `/api/assets`）
3. 确认分页：每页条数切换、页码点击、上一页/下一页正常
4. 确认搜索框过滤
5. 确认开始检测、导出Excel按钮可用
6. `npm run build` 打包后 `api_server.py` 指向 `dist/`，生产环境验证

## 风险与权衡

| 风险 | 影响 | 应对 |
|------|------|------|
| 重构期间 Flask API 改了 | 前端白写 | API 端点保持不动 |
| Element Plus 组件熟悉成本 | 进度慢 | 先照着现有功能搬，不挑战新用法 |
| DataTables 导出Excel 迁移 | 功能丢失 | Element Plus 暂无原生 Excel 导出，用 xlsx 库手动实现 |
| 演示数据 fallback | 看不到效果 | 重构初期仍保留 demo 数据逻辑 |
